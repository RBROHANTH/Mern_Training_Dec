1. Static Position (Default)

What it does: Elements are positioned according to the normal document flow.
Use case: This is the default behavior. You rarely need to set this explicitly.
Note: top, right, bottom, left, and z-index have NO effect on static elements.
position: static; /* Default - no special positioning */
I'm Static (Normal Flow)

2. Relative Position

What it does: Element is positioned relative to its NORMAL position. The original space is preserved in the document flow.
Use case: Small adjustments, or as a container for absolute children.
position: relative; top: 20px; left: 30px;

3. Absolute Position

What it does: Element is removed from normal flow and positioned relative to its nearest positioned ancestor (parent with position: relative/absolute/fixed).
Use case: Dropdowns, tooltips, overlays, badges on icons.
Important: If no positioned ancestor exists, it uses the document body!
/* Parent */ position: relative;
/* Child */ position: absolute; top: 20px; right: 20px;

I'm the parent with position: relative (orange dashed border)

I'm Absolute (top: 20px, right: 20px from parent)

4. Fixed Position

What it does: Element is removed from normal flow and positioned relative to the VIEWPORT. It stays in place even when you scroll!
Use case: Navigation bars, chat buttons, cookie banners, back-to-top buttons.
position: fixed; bottom: 20px; right: 20px;

5. Sticky Position

What it does: A hybrid! Element acts as relative until you scroll past a threshold, then it becomes fixed.
Use case: Table headers, section headers, navigation that sticks after scrolling.
Important: You MUST specify at least one of: top, right, bottom, or left!
position: sticky; top: 0;
